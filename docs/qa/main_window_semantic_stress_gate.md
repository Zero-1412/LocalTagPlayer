# 主界面语义压测门禁

## 目标

主界面真实窗口压测优先使用辅助树语义标签，不再依赖固定坐标。脚本用于长期复跑标签筛选、排序、本地媒体库路径切换和视频打开/返回入口。

## 失败分类

- `ui_state_wait`：目标属于当前 UI 状态应该出现，但等待后仍不可见，例如排序按钮、排序项、本地媒体库 root 或二级 chip 暂时不可点击。
- `list_visibility`：目标存在于列表或路径结果中，但当前滚动窗口没有把可操作项暴露出来，例如滚动后仍找不到视频播放按钮。
- `tag_expansion`：一级标签点击后没有展开可用二级 chip，说明标签展开状态或一级/二级归属需要复查。

## 门禁判断

- 先看 `gate.ok`：应用退出、窗口丢失或异常停止必须视为失败。
- 再看 `failureBuckets`：确认失败是否集中在 UI 等待、列表可见性或标签展开。
- 最后看 `failureDetails`：按失败类型、阶段和原因定位误报来源，避免把所有残余问题都归为“目标暂时不可见”。

## 复跑要求

- 涉及媒体库、标签筛选、排序方式、视频卡片入口或本地媒体库路径浏览的改动后，应至少跑一次语义压测短跑。
- 如果出现 `app_exit_or_window_lost` 或连续相同语义目标重复命中，应停止测试并优先分析应用状态或辅助树定位。
- 如果只有 `ui_state_wait` 或 `list_visibility`，先结合 `failureDetails` 判断是否为等待时间、滚动可见性或当前页面状态导致的误报，再决定是否修改业务代码。
