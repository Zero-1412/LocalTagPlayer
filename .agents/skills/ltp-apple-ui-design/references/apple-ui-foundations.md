## 1. 产品解释

Apple 式观感来自“目的、控制、熟悉、灵活、简洁、工艺和愉悦”的共同结果，不来自单一圆角、字体或透明度。

Local Tag Player 的目标情绪是平静、可靠、专注。核心视觉顺序始终是：

```text
当前上下文
-> 搜索与活动筛选
-> 视频内容
-> 维护动作
-> 装饰层次
```

设计必须让用户随时回答：当前在哪里、正在筛选什么、有多少结果、下一步能去哪、怎样返回。

## 2. 层级与材质

使用三层结构：

| 层级 | 用途 | 建议 |
| --- | --- | --- |
| 画布 | 媒体库、播放器背景 | 稳定深色，不做动态模糊 |
| 结构表面 | 左右侧栏、设置分组、详情区 | 较实、较深、弱描边，保证信息密度 |
| 浮动表面 | 菜单、popover、sheet、临时工具条 | 可轻量半透明、柔和阴影和高光边，但必须可降级为实色 |

约束：

- 不在两个浅透明表面上继续叠第三层透明表面。
- 不把视频缩略图大面积 blur 到背景；这会制造 GPU 成本和视觉噪声。
- 模态任务使用 scrim 聚焦；非阻塞侧栏不使用 scrim。
- 滚动内容与浮动 chrome 相交时，优先使用轻渐变或阴影提示，不依赖硬分割线堆叠。
- 大表面圆角不等于更高级。现有桌面侧栏可保持直边；浮层和卡片再使用圆角表达层级。

推荐几何基线：

```text
紧凑控件圆角: 8-10
搜索/工具表面圆角: 12-14
浮层/大卡片圆角: 16-20
图标按钮命中区: 至少 40x40，关键桌面工具优先 44-48 高
常规间距: 4 / 8 / 12 / 16 / 24 / 32
```

这些值是范围，不是要求全局机械替换。先沿用现有 token，再在单个组件族中收敛。

## 3. 排版与图标

- 默认使用 Windows 系统字体及项目现有中文 fallback，不引入 San Francisco 字体文件。
- 标题依靠字号、字重和行高共同建立层级，不只放大字号。
- 中文正文和密集标签默认 `letterSpacing: 0` 或非常轻微正值；不要照搬英文大标题的负字距。
- 文件名和标签允许省略，但必须提供 tooltip、详情或其它查看完整内容的路径。
- 数量、时长、进度和播放器时间使用稳定数字宽度，避免状态更新时左右抖动。
- 同一动作全应用使用同一图标。删除、打开位置、编辑标签、收藏、返回、筛选不得随页面换隐喻。
- 仅图标按钮必须提供 tooltip、焦点反馈和 Semantics label。

建议文字层级：

| 角色 | 典型范围 | 用途 |
| --- | --- | --- |
| 页面标题 | 20-24 / 700-800 | 页面身份 |
| 区域标题 | 16-19 / 700-800 | 分组与侧栏 |
| 主要正文 | 14-16 / 500-650 | 文件名、设置项 |
| 辅助正文 | 12-14 / 500-600 | 路径、说明、计数 |
| 状态角标 | 11-13 / 600-700 | 时长、来源、状态 |

必须在 100%、125%、150% 文字缩放下复核，不用固定高度截断重要设置说明。

## 4. 组件细节

### 按钮

- pointer-down 立即出现轻微缩放或颜色反馈，pointer-up 才提交动作。
- 主要按钮每个页面控制数量；紫色实心只用于最重要的下一步。
- 危险按钮使用清晰红色和具体动词，只有不可逆动作才要求确认。
- 可撤销动作优先 Snackbar 撤销，不重复弹确认框。

### 搜索与筛选

- 继续使用真实 `TextField`、controller 和统一 `onChanged` 链路。
- 搜索框只承担输入；活动筛选、结果数和进度放在相邻状态区域。
- 点击标签先更新选择态和结果，非关键计数随后刷新。
- 二级 folder 标签始终显示在所属一级标签上下文中。

### 视频卡片

- 缩略图是视觉中心，标题和必要状态为第二层，常用动作按 hover/focus 出现。
- hover 不能推动标题、改变网格列数或重新请求缩略图。
- 动态预览必须有意图延迟、可取消，并在播放或后台繁忙时守住资源预算。
- 卡片点击继续打开完整 filtered queue；装饰改造不能缩短队列。

### 菜单、Popover、Sheet 与 Dialog

- 菜单和 popover 从触发控件附近出现；Dialog 保持窗口中心。
- “更多”必须先展示菜单，不直接跳到某一动作。
- 进入与退出沿同一路径，退出通常比进入稍快。
- 设置中影响播放稳定性的选项必须有确认或撤销。

### 空、加载、完成和错误状态

- 空状态优先解释为什么为空以及下一步能做什么，不用装饰插画掩盖缺少动作。
- 加载显示当前阶段、进度或可取消性；避免无限 shimmer 覆盖大列表。
- 完成反馈简短，不阻断继续浏览。
- 错误紧邻失败对象，显示可理解原因和恢复入口，不只展示异常字符串。

## 5. 响应式与平台

```text
expanded: 常驻结构侧栏，支持鼠标深度工作流
medium: 可折叠侧栏或非模态 side sheet
compact: drawer / bottom sheet 与紧凑结果视图
```

- Windows 优先保留右键、hover、滚轮、侧键、键盘焦点和 tooltip。
- macOS 后续适配 Command 键、窗口工具栏习惯和系统材质，但共享业务 UI 不硬编码 macOS 行为。
- Linux 保留实色降级，避免依赖特定 compositor 的 blur 效果。
- 触摸设备不触发桌面 hover 动效；通过平台/指针能力区分，而不是依赖一次 hover 事件猜测。

## 6. 无障碍

- `MediaQuery.disableAnimations` 或 `accessibleNavigation` 启用时，去掉大位移、弹簧、视差和持续漂浮，保留短淡入、颜色和选中反馈。
- `MediaQuery.highContrast` 启用时，提高表面不透明度和描边对比，不能只依赖阴影分层。
- 焦点顺序跟随视觉阅读顺序；隐藏的 hover 操作仍要能通过键盘到达或从“更多”菜单到达。
- 颜色不是唯一状态编码；选中、错误、收藏和 missing 同时使用图标、文字或形状。
- 动态预览、自动隐藏和 timeout 不能让屏幕阅读器用户失去操作入口。
