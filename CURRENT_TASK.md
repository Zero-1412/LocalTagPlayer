## 活跃任务

### 2026-07-21 播放器文件重命名与标签入口解耦

- 目标：修复播放器详情中“文件名铅笔”和“继续添加”都打开标签编辑器的职责冲突。
- 当前状态：已完成。
- 文件名铅笔现在只打开“重命名文件”；弹窗只编辑 basename，扩展名只读保留，并明确提示标签应在下方标签区域维护。
- “添加标签 / 继续添加”仍只打开 manual 标签编辑器，不改变 folder/manual 来源规则。
- 物理改名经过 `FileSystemAdapter.renameFile`，拒绝覆盖同名文件；Store 只提交同目录 mutable path，保持同一 `videoId`、手动标签、收藏、播放记录和进度。
- 播放器先尝试不中断播放完成改名；仅在文件句柄占用时停止后端、重试并恢复原位置和播放/暂停状态。数据库提交失败会尝试恢复原文件名。

## 当前稳定基线

- 产品边界：Tag 驱动的本地视频发现播放器，不扩展字幕、音轨、逐帧或 A-B loop 等专业播放器能力。
- 数据边界：SQLite schema、`FilterQuery` / `TagQueryService`、filtered queue 内容与顺序、标签来源语义均未改变。
- 验证：233 项测试通过，3 项显式 benchmark 跳过；`flutter analyze` 与 Windows debug build 通过。1249×714 最新 Debug 实窗打开重命名弹窗并取消，文件名保持 `407247-1080p.mp4`，播放继续推进，返回媒体库恢复 11,170 条结果。
- 架构基线：`Architecture Baseline 0.5.50`。
